### Что делает UnoCSS в проекте

UnoCSS в этом проекте — это «слой утилитных классов», который генерирует CSS **ровно под те классы, которые есть в разметке и SASS/CSS**.
---Важно: после запуска локального сервера через npm run dev переменные, которые записаны в SASS через v(N) и в CSS через var(--N) сгенерируются только один раз. UNOCss не умеет отслеживать изменения SASS/CSS файлов. Но будет реагировать на каждое изменение в разметке. Эта особенность решается либо настройкой базовых переменных (105, 106 строки кода в конфиге) либо перезапуском сервака npm run dev.
Через него задаются отступы, размеры, сетка, флексы, позиционирование, типографика, z‑index и другие частые штуки.
Почти все числа (8, 16, 24 и т.д.) работают как **адаптивные значения**, которые плавно меняются под разные ширины экрана.

***

### Ключевая идея

1. В HTML и SASS ты используешь числа `N`:
    - классы `m-24`, `fz-18`, `top-40`, `gap-16` и т.п.
    - функция в SASS `v(24)`, `v(-16)`.
2. Конфиг UnoCSS пробегается по файлам, собирает все встреченные числа и генерирует под них набор переменных `--N` и `---N`.
3. Эти переменные пересчитываются под три зоны ширины: десктоп, ноутбук, мобильный, чтобы отступы/шрифты не были слишком большими или маленькими.
4. Все утилитные классы (`m-24`, `fz-18` и т.д.) берут значения только из этих переменных — за счёт этого всё становится «резиновым» без ручных формул в каждом месте.

***

### Брейкпоинты и префиксы

В конфиге заданы три основных точки, на которые завязана адаптивность.


| Имя | Ширина | Для чего |
| :-- | :-- | :-- |
| desktop | `1920.02px` | Значения считаются от макета 1920. |
| laptop | `1440.98px` | Зона ноутбуков и префикс `l:`. |
| mobile | `575.98px` | Мобилки и префикс `m:`. |

Префиксы:

- `l:` — применить класс только до ширины ноутбука (`max-width: 1440.98px`).
- `m:` — применить класс только до мобильной ширины (`max-width: 575.98px`).

Примеры:

```html
<div class="mt-40 l:mt-32 m:mt-24">
<div class="d-flex m:d-block">
```

### Как задавать размеры и отступы

#### В HTML — через классы

Примеры:

```html
<div class="m-24 px-16">...</div>
<h2 class="fz-24 mb-16">Заголовок</h2>
<div class="gap-24 d-flex">...</div>
```

- `m-24` → внешний отступ по всем сторонам.
- `px-16` → горизонтальные внутренние отступы.
- `fz-24` → размер шрифта.
- `gap-24` → расстояние между элементами в flex/grid.

Все эти числа (`24`, `16` и т.д.) попадают в генератор переменных и становятся адаптивными.

#### В SASS — через функцию `v(N)`

Когда нужен тот же размер, но в «обычном» CSS‑свойстве:

```sass
.section
  padding-top: v(40)
  margin-bottom: v(-24)  // отрицательное значение
```

- `v(40)` → подставится как `var(--40)` с теми же значениями, что и у `m-40`.
- `v(-24)` → подставится как `var(---24)`, то есть отрицательный отступ.

Так HTML‑утилиты и SASS‑классы используют одну и ту же шкалу размеров.

***

### Что генерируется «под капотом»

UnoCSS один раз собирает CSS‑переменные и медиазапросы.

Упрощённо:

```css
/* База: примерно для 1441–1920px */
:root {
  --8: 8px;
  --16: 16px;
  ---16: -16px;
}

/* Ширина ≥ 1920 */
@media (min-width: 1920.02px) {
  :root {
    --16: ...vw; /* от макета 1920 */
  }
}

/* Ширина ≤ 1440.98 */
@media (max-width: 1440.98px) {
  :root {
    --16: ...vw; /* от макета 1440 */
  }
}

/* Ширина ≤ 575.98 */
@media (max-width: 575.98px) {
  :root {
    --16: ...vw; /* от макета 375 */
  }
}
```

Дальше любые классы `m-16`, `fz-16`, `top-16` уже используют эти переменные.

### Группы классов, которые есть «из коробки»

Ниже — не полный список, а то, что нужно помнить в голове. Детали можно посмотреть в `uno.config.js`.

#### Отступы

- Общие: `m-N`, `p-N`.
- По сторонам: `mt-N`, `mb-N`, `ml-N`, `mr-N`, `pt-N`, `pb-N`, `px-N`, `py-N`, `mx-N`, `my-N`.
- Отрицательные: `-m-N`, `-mt-N`, `-mx-N` и т.п.

```html
<div class="mt-40 mb-24 mx-16">
<div class="-mt-20">
```


#### Размеры и шрифты

- Шрифт: `fz-N`.
- Ширина/высота в «шкале N»: `w-N`, `h-N`.
- Ширина/высота в процентах: `w-50p`, `h-100p`.

```html
<h1 class="fz-32 lh-base fw-700">...</h1>
<div class="w-50p h-100p">...</div>
```


#### Отступы между элементами

- `gap-N`, `gx-N`, `gy-N` для flex/grid.

```html
<ul class="d-flex flex-column gap-16">
```


#### Сетка по принципу 12 колонок

- `col-N` — ширина колонки в `N/12 * 100%`.
- `offset-N` — отступ слева по той же сетке.

```html
<div class="d-flex flex-wrap">
  <div class="col-8">...</div>
  <div class="col-4">...</div>
</div>
```


#### Display, flex, выравнивание

- Display: `d-none`, `d-block`, `d-flex`, `d-grid` и т.п.
- Flex‑направление: `flex-row`, `flex-column`, `flex-wrap`.
- Выравнивание: `justify-content-between`, `align-items-center`, `align-self-start` и др.

```html
<header class="d-flex justify-content-between align-items-center">
```


#### Позиция и координаты

- Положение: `position-relative`, `position-absolute`, `position-fixed`.
- Координаты: `top-N`, `left-N`, `-top-N`, `top-50p`, `inset-0`.

```html
<div class="position-absolute top-20 right-40">
<div class="position-fixed inset-0">
```


#### Остальное (вкратце)

- `overflow-*`, `visible` / `invisible`.
- `z-*` и `z--*` для z‑index.
- Радиусы: `rounded-N`, `rounded`, `rounded-full`, `rounded-circle`.
- Прозрачность: `opacity-15`, `opacity-50`, `opacity-100`.
- `object-cover`, `cursor-pointer`, `text-underline`, `whitespace-nowrap`, `max-w-full` и т.д.

***

### Адаптив: как комбинировать классы

Комбинация базового класса и префиксов:

```html
<h1 class="fz-32 l:fz-28 m:fz-24 lh-base fw-700">
  Заголовок
</h1>

<section class="px-40 py-60 m:px-20 m:py-30">
  ...
</section>
```

- Без префикса — «общий» вариант.
- `l:` и `m:` — уточнения для ноутбука и мобилки.

***

### Как добавить новые утилиты

Если нужно расширить набор классов:

1. Открыть `uno.config.js`.
2. В массиве `rules` добавить новое правило **в конец**.

Пример:

```js
// В конце rules
['text-shadow-soft', { 'text-shadow': '0 4px 20px rgba(0,0,0,.3)' }]
```

3. Запустить `npm run dev` и убедиться в отсутствии ошибок и корректной работе нового класса.

***

### Когда лезть в конфиг, а когда нет

Стоит трогать `uno.config.js`, если:

- не хватает какого‑то базового класса, который нужен во многих местах;
- нужно подправить базовую шкалу брейкпоинтов или расчёт размеров.

Не стоит:

- править уже существующие правила, если верстка проектов на них завязана;
- тащить туда бизнес‑логику (цвета бренда и т.п. лучше в SASS‑переменные).
